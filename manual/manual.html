2 Installation

After downloading the software package (e.g., exabayes-0.4.tar.gz), you will have to compile the source code. The following sections explain, how to do so. After compiling you obtain an executable called exabayes.

2.1 General compilation instructions

ExaBayes requires a relatively recent c/c++ compiler that supports c++11 features. ExaBayes is confirmed to work with

  1. GCC version 4.6 or greater
  2. Clang version 3.2 or greater

2.2 Comfort build

The preferred way of compiling ExaBayes is to use the small build.sh script. If you need to have more control over the build process, consider the next two sections.

Simple execution of the build script creates the sequential version of ExaBayes (called yggdrasil) and all pre- and post-processing tools in a build-folder. 

$ ./build.sh 
 [...] 
$ ls build 
asdsf  consense  credibleSet extractBips  obj-mpi  obj-seq  parser  postProcParam  yggdrasil

For building the (parallel) mpi-version of ExaBayes (executable is called exabayes), please provide the name of the mpi-C compiler (usually mpicc) and mpi-C++ compiler (mpicxx,mpiCC or mpic++) as environment variable arguments. For instance: 

$ ./build.sh CC=mpicc CXX=mpic++
[...]
$ ls build 
exabayes

2.3 Manual: Compiling the Sequential Version

To compile a sequential version of ExaBayes (called yggdrasil) and various pre- and post-processing tools, type 

$ ./configure && make

If you have a multi-core machine, you can speed up the build process by adding -j<numProc> to the make command.

2.4 Manual: Compiling the Parallel Version

For running ExaBayes on a cluster, you need to have a working installation of a library that implement the Message Passing Interface (MPI).

You have to use a slightly different configure call and inform the configure script about your mpi compiler wrapper. These are usually called mpicc (c compiler) and mpicxx/mpiCC or mpic++ (c++ compiler). For instance: 

$ ./configure --enable-mpi CC=mpicc CXX mpicxx
